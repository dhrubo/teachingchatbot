🧭 Navigation with a separate landing page

Since you chose a separate landing page (C), how should navigation work when the student starts a mission?

A

Landing at / — chat moves to /learn

/ = SARA dashboard (hero, stats, missions)
/learn or /chat = the chat tutoring view

Top nav bar: [SARA logo] [Learn] [Progress]
"Start mission" → navigates to /learn with topic pre-loaded

✅ Clean separation, real landing page
⚠️ Changes existing routing, logged-in redirect needs updating

B

Dashboard at /dashboard — chat stays at /

/dashboard = SARA hero + stats + mission + topic map
/ = current chat view (unchanged)
/chat/[id] = existing chats

Header nav switches between them
"Start mission" → navigates to / (new chat) with topic pre-loaded
✅ Minimal routing changes, backwards compatible
⚠️ Chat isn't under /chat

C

Landing overlays the empty chat

The SARA dashboard renders on / when no chat is active (replacing the current greeting component). Once a student starts a mission, the dashboard slides away and the chat UI takes over.

"Start mission" → transitions to chat view on the same page
✅ No routing changes, logged-in redirect still works
✅ Feels like a single-page app with smart states
⚠️ Less "separate page" than you might have wanted

I'd recommend A since you explicitly chose a separate page — it gives the cleanest separation. But C is much less work and keeps routing simple. What feels right?
