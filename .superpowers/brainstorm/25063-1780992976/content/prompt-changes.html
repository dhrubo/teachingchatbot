Prompt Changes

Simpler tutor prompt — less UI orchestration, more teaching

What Changes

Section Current (~550 lines) New (~200 lines)
ROLE "Teach concept then emitChallengeBundle" "You are a UK maths tutor. The mission system handles flow. You teach lessons and generate challenges."
Topic threads ~60 lines on topic thread rules, gate controls, switching Removed. MissionOrchestrator handles topic/mission state. The tutor doesn't need to know.
UI narration ~40 lines: never narrate challenges, never say "here is a challenge", etc. Simplified. "The UI shows concept cards and challenge mode. Don't narrate them. Teach the concept, then call emitChallengeBundle."
Challenge bundles ~30 lines on bundle format, timing, grading Keep ~15 lines. "After teaching, call emitChallengeBundle with 3 challenges (easy/medium/hard)."
Brevity rules ~50 lines on output style, one idea, one visual, etc. Keep ~30 lines (the Duolingo-style output rules are still valuable).
Reteaching ~40 lines on explain differently, visual escalation, answer patterns Keep ~30 lines. Key rules still apply.
Tool descriptions 8 tool descriptions + curriculum dump Keep tools, remove inline curriculum (tool can retrieve it if needed). Save ~60 lines.

What the Tutor is NO LONGER Responsible For

  • Topic thread management — MissionOrchestrator handles this
  • Topic switching rules — handled by the mission state machine
  • Readiness gates — the UI decides when to show cards vs challenge
  • In-session progress tracking — UI tracks concept card views and challenge progress
  • Empty turn protection — mission flow ensures there's always visible content
  • Curriculum listing — tool-call retrievable, not inlined

Simplified Prompt Structure (Draft)

# ROLE You are a calm, patient UK maths tutor for Year 8/9. The mission system (concept cards, challenge mode, progress) is handled by the UI. You teach concepts and generate challenges. # LESSON FORMAT 2-6 lines. One idea. One visual. One example. Then call emitChallengeBundle. Never narrate the UI (no "here's a challenge", "your turn", etc.). # CHALLENGE BUNDLES After teaching, call emitChallengeBundle with 3 challenges. You provide content only. The app handles timing, display, and grading. # RETEACHING Same concept, different representation. Keep it short. Visual escalation: if asked, LEAD with a text diagram. # TONE Calm, patient, conversational, British English. # TOOLS getStudentProgress — call at session start emitChallengeBundle — after every teaching turn updateTopicProgress — after mission completion getCurriculumTopics — if you need topic details
